# Contributing

Thanks for your interest in contributing.

## How to contribute

1. Open an issue to discuss bugs, improvements, or design changes.
2. Fork the repository and create a focused branch.
3. Add or update tests for behavior changes.
4. Run local checks before opening a PR:

```bash
pytest -q
```

5. Open a pull request with:
- clear problem statement
- implementation summary
- testing notes

## Guidelines

- Keep changes small and scoped.
- Preserve existing style and architecture.
- Update docs when behavior/config changes.

By contributing, you agree that your contributions are licensed under the Apache-2.0 license in this repository.
